# agent-docs

Auto-discover and index library documentation for AI coding agents.

> **Inspired by Vercel's research**: [AGENTS.md outperforms skills in agent evals](https://vercel.com/blog/agents-md-outperforms-skills-in-our-agent-evals). Vercel found that AI skills auto-discovery fails 56% of the time, while static documentation references in AGENTS.md achieved 100% reliability. agent-docs automates building that static index.

## The problem

AI coding agents (Claude Code, Cursor, Copilot, Codex) don't reliably discover library documentation on their own. Tools like Context7, skills.sh, and llms.txt all require the agent to **decide** to look up docs. In practice, agents skip this step more than half the time, leading to hallucinated APIs and outdated code patterns.

## The solution

`agent-docs` scans your project dependencies, discovers documentation sources for each library, and generates an index file referenced from your CLAUDE.md / AGENTS.md. The agent always sees the pointer in context and knows exactly where to find docs — no discovery needed.

## Quick start

```bash
npx agent-docs init
```

This will:
1. Read your `package.json` dependencies
2. For each dependency, check for: llms.txt, MCP endpoints, Context7 docs
3. Generate `.agent-docs/index.md` with pointers to all discovered sources
4. Inject a reference into your CLAUDE.md / AGENTS.md

## Usage

### CLI

```bash
npx agent-docs init                  # Full scan + generate + inject pointer
npx agent-docs update                # Re-scan (cache-aware, fast)
npx agent-docs scan                  # Dry run — preview output
npx agent-docs add humanizer --type skill --url https://skills.sh/humanizer
npx agent-docs config                # View configuration
```

### MCP Server

For AI agents that support MCP, add to your `.mcp.json`:

```json
{
  "mcpServers": {
    "agent-docs": {
      "command": "npx",
      "args": ["agent-docs", "mcp"]
    }
  }
}
```

The MCP server exposes: `init`, `update`, `add_resource`, `get_index`.

### Auto-update on install

Add to your `package.json`:

```json
{
  "scripts": {
    "postinstall": "npx agent-docs update --quiet"
  }
}
```

## What it discovers

For each dependency, agent-docs checks (in order):

| Source | How | What gets indexed |
|--------|-----|-------------------|
| **llms.txt** | `HEAD <homepage>/llms.txt` | URL to the llms.txt file |
| **MCP** | Curated registry of known MCP endpoints | MCP endpoint URL or install command |
| **Context7** | `GET context7.com/api/v2/libs/search` | Context7 library identifier |
| **Homepage** | npm registry metadata (fallback) | Documentation URL |

## Manual resources

Not everything maps to a dependency. Add skills, conventions, or docs that aren't tied to imports:

```bash
npx agent-docs add humanizer --type skill --url https://skills.sh/humanizer
npx agent-docs add "design-system" --type docs --url https://company.com/design/llms.txt
npx agent-docs add "coding-conventions" --type file --path ./docs/conventions.md
```

## Generated output

### `.agent-docs/index.md`

```markdown
# Documentation Index
> Auto-generated by agent-docs. Do not edit manually.

## Dependencies

### react
- **llms.txt**: https://react.dev/llms.txt
- **Context7**: `/facebook/react`

### better-auth
- **MCP**: endpoint `https://mcp.better-auth.com/mcp`
- **llms.txt**: https://better-auth.com/llms.txt

### zod
- **Context7**: `/colinhacks/zod`
- **Docs**: https://zod.dev

## Resources

### humanizer
- **Skill**: https://skills.sh/humanizer
```

### Agent file injection

agent-docs injects a pointer section into your agent instructions file:

```markdown
<!-- agent-docs:start -->
## Library Documentation
Before writing or modifying code that uses a project dependency, read `.agent-docs/index.md`
for up-to-date documentation sources.
<!-- agent-docs:end -->
```

**Smart detection**: If your `CLAUDE.md` references `@AGENTS.md`, agent-docs follows the reference and injects into `AGENTS.md`. Detection order: `CLAUDE.md` → `AGENTS.md` → `.cursorrules` → `.github/copilot-instructions.md`. Creates `AGENTS.md` if none exist.

## Configuration

Stored in `.agent-docs/config.json`:

```json
{
  "resolvers": {
    "llmsTxt": true,
    "context7": true,
    "mcp": true
  },
  "concurrency": 10,
  "exclude": ["@types/*"],
  "resources": [],
  "context7ApiKey": "ctx7sk_..."
}
```

## What to commit

- `.agent-docs/index.md` — commit this (so the AI has the index without running the tool)
- `.agent-docs/config.json` — commit this (shared config)
- `.agent-docs/.cache/` — add to `.gitignore`

## How it works

```
package.json
    ↓
parseDependencies()
    ↓
┌─────────────────────────────────────────┐
│  For each dependency (parallel, limit 10) │
│                                           │
│  1. npm registry → homepage, repo URL     │
│  2. HEAD homepage/llms.txt → found?       │
│  3. known-mcps.json → MCP endpoint?       │
│  4. Context7 API → indexed?               │
│  5. fallback → homepage as docs URL       │
└─────────────────────────────────────────┘
    ↓
generateIndexContent()
    ↓
.agent-docs/index.md + CLAUDE.md pointer
```
